# Django Blog

Простой блог на Django с возможностью создания, редактирования, удаления и просмотра статей. Пользователи могут оставлять комментарии, а администраторы — управлять всем контентом через админ-панель.

## Функционал

- Просмотр списка статей с пагинацией (по 5 статей на странице)
- Просмотр деталей статьи с комментариями
- Добавление комментариев к статьям
- Поиск по статьям
- Фильтрация статей по категориям
- Адаптивный дизайн (Bootstrap 5)
- Админ-панель для управления статьями, комментариями и категориями
- Загрузка изображений к статьям

## Требования

- Python 3.8+
- pip

## Установка и запуск

### 1. Клонирование репозитория (если необходимо)

```bash
git clone <url-репозитория>
cd <папка-проекта>
```

### 2. Создание виртуального окружения (рекомендуется)

```bash
py -m venv venv
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Применение миграций

```bash
py manage.py migrate
```

### 5. Создание суперпользователя (администратора)

```bash
py manage.py createsuperuser
```

Следуйте инструкциям в терминале для ввода имени пользователя, email и пароля.

### 6. Запуск сервера разработки

```bash
py manage.py runserver
```

Сервер запустится по адресу: **http://127.0.0.1:8000/**

### 7. Доступ к админ-панели

Откройте в браузере: **http://127.0.0.1:8000/admin/**

Введите данные суперпользователя, созданного на шаге 5.

## Основные страницы

- **Главная**: `/` — список всех статей с пагинацией
- **Детали статьи**: `/post/<id>/` — страница статьи с комментариями
- **Поиск**: `/?search=<запрос>` — поиск по статьям
- **Фильтр по категории**: `/?category=<slug>` — фильтрация по категории
- **Админ-панель**: `/admin/` — управление контентом

## Дополнительная информация

- Изображения статей сохраняются в папку `media/`
- Для продакшена настройте `DEBUG = False` и укажите правильные `ALLOWED_HOSTS` в `settings.py`
- Настройте веб-сервер (Nginx, Apache) и WSGI-сервер (Gunicorn, uWSGI) для развертывания
